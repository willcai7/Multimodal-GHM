# Figures

All figures for the Multimodal Generative Hierarchical Model are available in the `output/` directory. They may differ slightly from those in our paper (`https://arxiv.org/abs/2501.04641`) due to prior infrastructure incidents on our lab cluster that led to data loss and required re-running experiments and regenerating plots.

## Regeneration

From the repository root, regenerate figure inputs and PDFs with:

```bash
python scripts/download_ckpt.py
python scripts/download_data.py
bash scripts/reproduce_eval.sh
bash scripts/reproduce_figures.sh
```

The GHM JSON inputs are written to `figures/data/ghm-data/`. The ImageNet Fig. 7 tensor is written to `figures/data/imagenet-data/similarities_targets.pt`. Rendered PDFs are written to `figures/output/`.

The root `README.md` contains the complete display-item map linking each figure to its input data script, plotting notebook, and expected PDF output.

Fig. 1, Fig. 3, and Fig. 4 are schematic illustration examples authored manually, e.g. in Adobe Illustrator/Keynote. They are not experimental results and are not regenerated by the training, evaluation, or notebook wrappers.

`scripts/reproduce_figures.sh` executes the active plotting notebooks:

- `Fig2-taskRisks.ipynb` -> `Fig2-a-*` through `Fig2-d-*`.
- `Fig56-ZSC-m.ipynb` -> `Fig5-*` and `Fig6-*`.
- `Fig7-imagenet.ipynb` -> `Fig7-a-*` through `Fig7-c-*`.
- `Fig8-OODRisks.ipynb` -> `Fig8-a-*` through `Fig8-d-*`.
- `Fig9-OODrisks2.ipynb` -> `Fig9-a-*` and `Fig9-b-*`, using modality-specific OOD sweeps where one probability is fixed at 0.20 and the other modality is varied.
